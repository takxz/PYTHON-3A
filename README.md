Un framework est une une architecture imposée, composant préécrits, règles d'organisation de code, cycle d'éxécution contrôlé par le framework lui-même.

Processus : programme en cours d'exécution, qui a son propre espace mémoire, ses propres ressources et est totalement isolé. Si un crash, les autres ne sont pas impactés.

Thread est un flux d'exécution à l'intérieur du processus, même mémoire et même ressources. Nécessite la gestion rigoureuse de la concurrence
